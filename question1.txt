کتابخونه : 
کتابخانه فقط مجموعه ای از api ها(مثله کلاس ها,متدها,ماژول ها,...) هست که اونا رو به کدخودمون اضافه میکنیم,
 و برای ساده تر شدن کد نویسی از اونا استفاده میکنیم مثال خیلی ساده اون کتابخانه ی Math توی زبان C هست که توی 
 اکثر زبان های برنامه نویسی هم وجود داره, 
 مثلا اگه بخواهیم لگاریتم ی عدد رو بدست بیاریم دیگه نیازی به دونستن الگوریتم بدست آوردن اون نیست خیلی ساده از 
 تابع کلاس Math استفاده میکنیم.
 
 

فریم ورک :  
 فریم ورک ی مفهوم پیچیده تره که درواقع به این صورته که وقتی API ها در یک مجموعه واحد برای یک هدف مشخصی قرار میگیرن و 
 الگو های خاصی رو پیروی کنن (مثله مدل MVC) دیگه فراتر از API هستن و بهش میگن فریم ورک ,
 درواقع فریم ورک مثله یک چارچوب واستخوان هست که نیاز هست
 برنامه نویس با نوشتن کد اونو کامل کنه و این کدها همان گوشت برای استخوان(فریم ورک) هست.
 
 تفاوت فریم ورک و کتابخانه : 
 درواقع این برنامه نویس هست که کتابخانه های آماده رو صدا میزنه درحالیکه فریم ی چارچوب آماده هست
 که برنامه نویس رو صدا میزنه برای نوشتن کدهای خودش و کامل کردن اون.
